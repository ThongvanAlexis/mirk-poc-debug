## Hypothesis

After the Phase 3.1 fix in `_FogPainter.paint()` — derive `(uOffsetX, uOffsetY) = ((camera.pixelOrigin.x / size.width) % 1.0, (camera.pixelOrigin.y / size.height) % 1.0)` from the same `MapCamera` snapshot already captured in `FogLayer.build()` and forward to `shaderRenderer.render(offset: ...)` — the same-Canvas fog now translates with the camera during pan.

Re-stated against the original POC core value: the fog-of-war stays perfectly locked to the map during pan, zoom, and combined gestures on a sideloaded iOS build. **Confirmed-after-fix** means the Phase 3 hypothesis is reinstated and the POC ports back to MirkFall with the Plan 03.1-02 fix included; **denied-final** means the same-Canvas architecture is unfixable in `flutter_map 7.0.2` and the POC produces the formal architectural counter-evidence (the Phase 3.2 alternative-renderer pivot becomes the only remaining option, and is itself a deferred-decision per CONTEXT §Fix-failure contingency).

### NEW: Phase 3.1 diagnostic stream (FOG-10) — `infrastructure.mirk.fog_transform`

The Plan 03.1-01 `FogTransformLogger` emits 1-Hz JSONL rollups capturing per-paint `(canvas.getTransform()[12..13], camera.pixelOrigin, camera.center, appliedUOffset)`. Post-walk grep-correlates this stream against `frame_delta` + `sdf` rollups on per-second `epochSecond` boundaries. Three correlation patterns expected:

- **CONFIRMED-AFTER-FIX:** `pixelOriginX/Y` track the user's pan trajectory monotonically; `uOffsetX/Y` modulo-1.0 mirror that movement (small jumps as the modulo wraps); `canvasTx/Ty` flat at 0 (per RESEARCH §Pitfall D — Canvas at identity inside `MobileLayerTransformer` at rotation=0). The diff between `pixelOrigin` and `appliedUOffset * size` confirms the conversion math is correct in the binary that was just walked.
- **DENIED-FINAL pattern #1:** `pixelOriginX/Y` track but `appliedUOffset` is `(0,0)` — fix did not land in the IPA (CI/sideload regression).
- **DENIED-FINAL pattern #2:** `pixelOriginX/Y` track AND `appliedUOffset` tracks AND fog still appears static — root cause is downstream of the shader uniform (e.g. shader code precision-loss at very large pixelOrigin magnitudes; possibility #1's underlying mechanism is deeper than the shader-uniform-bypass research diagnosed). This pattern triggers Phase 3.2 conversation.
