<objective>
Close the SHADER-MODULO-WRAP gap from `03.1-FALSIFICATION.md` (observations 2 + 3, "seed changing" shimmer + intermittent partial pan tracking). The Plan 03.1-02 fix derived `(uOffsetX, uOffsetY) = (camera.pixelOrigin.x / size.width % 1.0, camera.pixelOrigin.y / size.height % 1.0)` at the Dart call site — this introduced a per-paint single-frame discontinuity at the modulo wrap (Finding 3 of the post-fix walk: uOffsetX sweeps `0.005..0.99` 5-10× per 1-Hz rollup during gestures, producing the visible noise-seam shimmer).

This plan moves the modulo INSIDE the fragment shader: rename slot 3..4 from `uOffset` to `uPixelOrigin`, forward the full-precision world-pixel origin to the shader, and apply `fract(uPixelOrigin / uResolution)` per-fragment so each pixel computes its noise sample coordinate with floating-point precision local to its own UV. The discontinuity disappears because consecutive paints no longer hop across a Dart-side modulo wrap — the shader sees a continuously increasing pixelOrigin and the per-fragment `fract()` smoothly wraps without producing a global single-frame jump.

Purpose: this is the load-bearing architectural change for the post-iterating-walk re-test. Per the 03.1-FALSIFICATION.md hypothesis triage table, observations 2 + 3 collapse onto this single fix axis; if it works, the noise pattern stays stable across frames during pan and the developer's "seed changing" failure mode resolves. Plan 03.1-05 (CANVAS-FRAME-ALIGNMENT) layers the second-axis fix on top before the walk.

Output: shader uniform renamed end-to-end (atmospheric_fog.frag + FogShaderUniforms + RecordingFogShaderRenderer + _FogPainter + FOG-09 test); FOG-11 GREEN keystone behavioural test; updated REQUIREMENTS.md + ROADMAP.md.
</objective>
