## Decisions Made

- **Task 1 deferred the FogLayer mount-site `fogTransformLogger:` arg until Task 2.** Plan 03.1-02 Task 1 prescribed adding the arg to `lib/presentation/screens/map_screen.dart` lines 282-288 alongside the lifecycle wiring; but `FogLayer`'s constructor doesn't accept the field until Task 2 lands. Adding the arg in Task 1 would have failed `flutter analyze --fatal-infos lib/presentation/screens/map_screen.dart` mid-task — Task 1's `<verify>` block runs that command. The split keeps each task independently green and follows the plan's `<action>` step 3 ordering ("FogLayer parameter list once Task 2 lands"). Net effect: identical post-Task-2 state.
- **`import 'package:mirk_poc_debug/infrastructure/mirk/fog_transform_logger.dart'` not added to `lib/presentation/screens/map_screen.dart`.** The plan prescribed adding it; but `map_screen.dart` only references the logger via `widget.services.fogTransformLogger` (no direct symbol). Adding the import would trigger `unused_import` under `--fatal-infos`. The plan's intent (lifecycle wiring) is met without the dead import.
- **`runAsync(...)` budget for SDF cache resolution in FOG-09 test** — 30 × 20 ms = 600 ms upper bound. The painter's `if (sdfImage == null) return;` guard short-circuits paint() on first frame because `ui.decodeImageFromPixels` needs a real platform tick. Budget is generous; SDF cache resolves in <50 ms in practice.
- **`_MockCanvas` overrides only the Canvas methods `_FogPainter.paint()` actually calls** (save/restore/clipPath/drawRect/getTransform). Everything else throws via the `Fake` base — if the painter ever grows a new Canvas call (e.g. drawCircle for a debug overlay), the test fails fast rather than silently masking the regression. Same idiom as the Flutter SDK's own `custom_paint_test.dart`.
