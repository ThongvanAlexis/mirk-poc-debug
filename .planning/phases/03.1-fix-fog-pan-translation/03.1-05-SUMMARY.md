int _hash(List<RevealDisc> discs, MirkViewportBbox viewport) {
  final discHashes = discs
      .map((d) {
        final qlat = (d.lat * _spatialQuantisationFactor).round();
        final qlon = (d.lon * _spatialQuantisationFactor).round();
        final qrad = (d.radiusMeters * _radiusQuantisationFactor).round();
        return Object.hash(qlat, qlon, qrad);
      })
      .toList(growable: false);
  // PERF-08 (Plan 03.1-05) — viewport bbox quantised to ~11 m granularity ...
  return Object.hash(_quantiseBbox(viewport), Object.hashAll(discHashes), discs.length);

int _quantiseBbox(MirkViewportBbox v) {
  final qs = (v.south * _bboxQuantisationFactor).round();
  final qw = (v.west * _bboxQuantisationFactor).round();
  final qn = (v.north * _bboxQuantisationFactor).round();
  final qe = (v.east * _bboxQuantisationFactor).round();
  return Object.hash(qs, qw, qn, qe);
}

## Decisions Made

- **Option (a) over option (b) for CANVAS-FRAME-ALIGNMENT.** Per CONTEXT.md decision, chose to pre-shift the clip path via `canvas.getTransform()[12..13]` subtraction in `_FogPainter.paint()` rather than move the blue-dot rendering inside FogLayer's Canvas. Less invasive — the blue-dot CircleLayer stays in its own UNTRANSLATED Canvas; only the painter changes.
- **Single-snapshot at the matrix level.** `canvas.getTransform()` called EXACTLY ONCE per paint, mirroring FOG-07's single-MapCamera-snapshot lock. The same Float64List is reused for both the clip-path correction AND the fogTransformLogger emission (no second allocation, no second read).
- **Viewport bbox quantisation at 1e4 (1e-4 lat/lon ≈ 11 m).** Sized between per-paint micro-drift (~1e-7 lat/lon during pan) and any real GPS-fix-driven jump (1 m → ~9e-6 lat/lon). Conservative: cache rebuilds only when the viewport meaningfully shifts.
- **Zero new seams on SdfRebuildLogger for PERF-08 testing.** Reused the existing `Logger.root.onRecord` Stream pattern verbatim from `sdf_rebuild_logger_test.dart` Test 1; added a JSON-prefix filter `'{'` to separate rollup lines from the donor RevealedSdfBuilder's non-JSON status logs (both share logger name `'infrastructure.mirk.sdf'`).
- **GoRouter context.push + explicit IconButton(Icons.arrow_back) for UX-01.** Material's `automaticallyImplyLeading` would insert an `Icons.arrow_back` IconButton wired to `Navigator.maybePop`, but `Navigator.pop` does NOT pop a GoRouter route reached via `context.push` (GoRouter requires `context.pop()`). Per CLAUDE.md GoRouter rule: 'if the word retour has meaning in UX → push'.

**1. [Rule 1 - Bug] Sub-pixel tolerance in FOG-12 widget test (1e-3 instead of `kPocCanvasTransformEpsilon` = 1e-6)**

- **Found during:** Task 1 (CANVAS-FRAME-ALIGNMENT)
- **Issue:** Initial Y-axis assertion `(shiftedBounds.top - (identityBounds.top - (-44.198))).abs() < 1e-6` failed with actual delta `~1.86e-6`. dart:ui's `Path.combine(PathOperation.difference, worldPath, holesPath)` introduces ~1e-6 float noise per axis on non-axis-aligned shapes (the disc oval); `kPocCanvasTransformEpsilon = 1e-6` was sized for matrix-element comparisons + raw pixel-origin deltas (~411 raw pixels at zoom 13), not path-bound deltas.
- **Fix:** Replaced `kPocCanvasTransformEpsilon` with a local `const subPixelTolerance = 1e-3` (millipixel — well below visual divergence threshold but well above float-precision noise). Removed the now-unused `kPocCanvasTransformEpsilon` import. Added explanatory docstring.
