## Decisions Made

- **FogTransformLogger structurally mirrors SdfRebuildLogger verbatim** — confirmed handshake. Same constructor seam (`rollupInterval` defaults to `kPocFogTransformLogRollupSeconds`), same `start`/`stop`/`_emitRollup` shape, same `Logger.info` JSONL emission. The only structural deviations are: (1) buffer is `List<_FogTransformSample>` not `List<double>` because the sample is a 9-field record; (2) `recordPaint` carries 4 named args (canvasTransform/cameraPixelOrigin/cameraCenter/appliedUOffset) instead of `recordRebuild`'s 3; (3) `_emitRollup` materialises 8 sorted column-views and emits 26 keys (vs SdfRebuildLogger's 7).
- **min/median/max instead of median/p95/max** — p95 has no semantic meaning for spatial coordinates; the bounding values (min, max) are what diagnose whether `pixelOriginX` is moving during a pan or staying static.
- **toStringAsFixed(6) string values in JSONL** (not numeric doubles) — preserves the 6-decimal precision visibly. Aligned with `kPocCanvasTransformEpsilon` (1e-6) so a future `grep '"pixelOriginXMin":"[0-9.]*"'` reads at exactly the test-epsilon resolution. Plan 03.1-02 (consumer) and Plan 03.1-03 (walk evidence) both rely on this string representation.
- **`computeStats` exposed as `static @visibleForTesting`** rather than baked inline into `_emitRollup` — same testability pattern as `FrameDeltaProbe.debugRecordRawDelta`. The 4 behavioural tests in this plan don't call it directly, but Plan 03.1-02 may want unit-level stat assertions, and the helper makes that trivial.
- **`kPocFogTransformBufferMaxSamples` reference count is 2 in `fog_transform_logger.dart`** (1 in docstring, 1 in the FIFO drop loop) — plan's `done` criterion said "returns 1" but the docstring reference improves discoverability. The plan's intent ("FIFO drop loop must reference the constant") is met; the docstring reference is informational and aligns with the `SdfRebuildLogger` precedent of explaining buffer-cap discipline in the class docstring.
