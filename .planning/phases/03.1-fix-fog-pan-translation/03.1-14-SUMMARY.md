## Decisions Made

- **C-b chosen over C-a / C-c per CONTEXT.md C2' decisions.** C-a (`uResolution` semantic shift) was rejected as brittle (uResolution is also used for aspect-ratio + screen-mapping; semantic shift would force unrelated changes). C-c (Dart-only zoom-snap interpolation) was held as fallback (smooths the SNAP cosmetically without anchoring cells; symptom-targeting only). C-b lands cleanly with no slot-allocation conflicts; no fallback was needed.
- **`kPocFogReferenceZoom = 13.0` baseline.** Walks #1-#5 default-zoom regime; the FOG-17 noise pattern's character was visually verified at this zoom across all walks. At this zoom `uZoomScale = pow(2, 0) = 1.0` → shader formula reduces to pre-fix `worldPx / kNoiseTilePx`. Pixel-perfect MIRL visual-identity preservation at the reference zoom.
- **MIRL visual-identity-preservation rule honoured (CLAUDE.md `# MIRL solution` updated 2026-05-04).** Shader edits authorised because (a) `uZoomScale` is an ABI uniform consumed consistently by both conforming shaders (production + debug); (b) at uZoomScale = 1.0 the noise sampling is bit-identical to pre-fix; (c) at other zooms, the noise pattern's character at any settled (zoom, position) post-fix matches pre-fix at SOME equivalent (zoom, position) — the world-anchoring shifts WHICH samples are visible, not HOW the noise looks. Passes the Shadertoy-equivalence test.
- **B3 task structure (diagnostic + fix bundled in one plan).** Task A (DEBUG-03) and Task B (FOG-19) ship as separate atomic commits in one plan; Walk #6 simultaneously validates the C-b fix AND quantitatively measures any residual drift via the unique-numbered cells. Lower coupling cost than B1 (fix-first, risks shipping wrong fix) or B2 (measure-first then fix, adds an extra walk).
