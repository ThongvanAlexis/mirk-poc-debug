## Verbatim shader uniform rename — `assets/shaders/atmospheric_fog.frag`

**Before (line 62-65 region):**
```glsl
// World pan offset (in noise UV units). Lets the fog drift with the
// MapLibre camera so static fixtures don't appear to move when the user
// only pans. Slot 3..4.
uniform vec2  uOffset;
```

**After:**
```glsl
// World pixel-origin from MapCamera.pixelOrigin (full-precision world
// pixel units). At zoom 13 magnitudes are ~1e6; at zoom 15 ~4e6. The
// per-fragment `fract(uPixelOrigin / uResolution)` below moves the
// modulo wrap from CPU to GPU — pre-Plan-03.1-04 the Dart call site
// applied `% 1.0` and produced visible single-frame discontinuities at
// the wrap boundary (03.1-FALSIFICATION.md Finding 3 — the modulo wrap
// produced the "seed changing" shimmer 5-10× per second during gesture).
// Slot 3..4 (unchanged).
uniform vec2  uPixelOrigin;
```

**Before (line 256-258 region — the load-bearing change):**
```glsl
    // Apply world pan to the noise UV space (NOT to fragUv — fragUv is
    // screen-local for SDF sampling).
    vec2 noiseUv = fragUv + uOffset;
```

**After:**
```glsl
    // Apply world pan to the noise UV space (NOT to fragUv — fragUv is
    // screen-local for SDF sampling). The `fract()` is the per-fragment
    // modulo that pre-Plan-03.1-04 was applied at the Dart call site
    // (and produced the per-paint global discontinuity documented in
    // 03.1-FALSIFICATION.md Finding 3). Each fragment now computes its
    // own modulo in floating-point precision local to its UV.
    vec2 noiseUv = fragUv + fract(uPixelOrigin / uResolution);
```

This is the architectural fix: the Dart call site no longer applies any transformation to `pixelOrigin`; each fragment computes its own `fract()` on its own UV.

## Decisions Made

- **Modulo location: GPU per-fragment, not CPU per-paint.** The fragment shader's `fract(uPixelOrigin / uResolution)` runs at every pixel, every frame, with no consecutive-paint dependence. The Dart call site forwards `pixelOrigin` verbatim. Each pixel's noise sample coordinate is now continuous across consecutive paints because the global Dart-side modulo wrap that produced the discontinuity has been eliminated.
- **JSONL field names preserved.** The 03.1-03 walk's `.uat-tmp/fog_transform.jsonl` extracts use the keys `uOffsetXMin/Median/Max` and `uOffsetYMin/Median/Max`. Renaming these for "consistency" with the new uniform name would break the post-walk grep tooling already in use. The values now span the higher-magnitude range (~1e6..4e6) instead of the modulo fraction (0..1); post-walk tooling reads the higher values directly without any code change. The `appliedUOffset` recordPaint() parameter NAME is preserved for the same reason.
- **FOG-04..07 retained at `Falsified-in-production`.** This plan does NOT flip them to Complete. The SHADER-MODULO-WRAP fix is necessary but not sufficient; CANVAS-FRAME-ALIGNMENT (Plan 03.1-05) must also land before the iPhone walk in Plan 03.1-06 re-validates them. Plan 03.1-06's verdict is what flips FOG-04..07 to Complete (or strengthens them to Falsified-final).
- **FOG-11 magnitude assertion as defence-in-depth.** The consecutive-delta assertion (`dx < kPocFogSmoothCoordinateMaxDelta`) alone could pass vacuously against a future regression that re-introduces a Dart-side modulo (the deltas in normalised [0, 1) regime would still mostly be small, except at the wrap boundary which the test's 10-step pan might or might not cross). The second assertion (`captured.last.$1 > 100`) ensures the captured magnitude is in the raw-pixel regime regardless of whether any modulo wrap was crossed during the test pan.

## FOG-11 vs pre-Plan-03.1-04 HEAD verification

The FOG-11 test would FAIL against pre-Plan-03.1-04 HEAD (`5c63197`):
- The `dx < kPocFogSmoothCoordinateMaxDelta` consecutive-delta assertion: under the pre-fix Dart-side `(pixelOrigin / size) % 1.0`, captured values live in [0, 1). Most consecutive deltas are sub-pixel ratios (also [0, 1)); the wrap boundary produces a ~0.998 jump for any pan crossing it. 0.998 IS less than 1e3 (the threshold is 1000 raw pixels), so this assertion alone is INSUFFICIENT to fail pre-fix — the threshold is calibrated for the post-fix raw-pixel regime.
- The defence-in-depth `captured.last.$1 > 100` magnitude assertion: under the pre-fix modulo, captured values are bounded in [0, 1) by construction. The assertion `< 1 < 100` FAILS, catching any Dart-side modulo regression mechanically.

The combination is what makes FOG-11 catch the regression: the consecutive-delta assertion catches the smoothness regression in raw-pixel-regime, the magnitude assertion catches the units regression. Neither alone is sufficient; both together are.

This was verified by mental simulation rather than checkout-against-HEAD; the actual on-iPhone walk in Plan 03.1-06 will confirm the developer's "seed changing" failure mode is resolved.
