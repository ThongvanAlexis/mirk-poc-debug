### Step 2 ‚Äî Why gradient-noise FBM cannot reach ZERO

The fbm3 chain is:
```
fbm3(p) = 0.5 ¬∑ noise3(p)
        + 0.25 ¬∑ noise3(p ¬∑ M_1 + V_1)
        + 0.125 ¬∑ noise3((p ¬∑ M_1 + V_1) ¬∑ M_2 + V_2)
```

with `M_1 = 2.03¬∑I`, `M_2 = 2.05¬∑I`, `V_1 = (13.7, 7.3, 5.1)`, `V_2 = (-11.1, 17.9, 3.3)`. At a wrap event, `p` shifts by integer-cell vector `V = (-4, 0, 0)` (= `kPocFogIntegerWrapPeriodMeters / kPocFogNoiseTilePxMeters` = 4096/1024).

- Octave 1: `noise3(p + V)` is bit-identical because hash3 has period 1 in each axis and V is integer-cell. ‚úì
- Octave 2: `noise3((p + V) ¬∑ M_1 + V_1) = noise3(p ¬∑ M_1 + V_1 + M_1 ¬∑ V) = noise3(... + (-8.12, 0, 0))`. The shift `M_1 ¬∑ V = (-8.12, 0, 0)` has fractional part `0.12` ‚Äî NON-zero. Octave 2 produces a deterministic phase shift.
- Octave 3: similarly, `M_1 ¬∑ M_2 ¬∑ V = (-16.646, 0, 0)`, fractional part `0.646`. Octave 3 produces another deterministic phase shift.

The fractional parts are CONSTANT (matrices and V are constants), so the per-wrap discontinuity is INVARIANT. Bounded analytically by `0.25 ¬∑ max||‚àánoise3|| ¬∑ 0.12 + 0.125 ¬∑ max||‚àánoise3|| ¬∑ 0.646 ‚âà 0.188 < 0.20`. **This bound cannot reach 0** because the matrices `M_1`, `M_2` are uniform-scaling (NOT integer-multiple) and Octave 2 + 3 sample positions fractionally shift at every integer-cell wrap.

### Step 3 ‚Äî Worley periodic primitive achieves ZERO

Worley (cellular) noise samples a regular grid of feature points and returns the distance to the nearest one (F1). In the periodic variant:
```
worleyPeriodic(p) {
  // Period in each axis = N (in normalised noise-UV space; 1.0 in our convention because
  // we sample noiseUv = worldMeters / kPocFogNoiseTilePxMeters, so 1.0 in noiseUv == 1024 m == one cell).
  vec2 cell = floor(p);
  vec2 fract_p = fract(p);
  float minDist = 1.0e6;
  for (int dy = -1; dy <= 1; dy++) {
    for (int dx = -1; dx <= 1; dx++) {
      // Toroidal wrap: feature point at (cell + offset) modulo period N.
      vec2 neighborCell = mod(cell + vec2(dx, dy), N);
      vec2 jitter = hash2(neighborCell) * kPocFogWorleyJitterAmplitude;
      vec2 featurePoint = vec2(dx, dy) + jitter; // relative to fract_p
      float d = length(fract_p - featurePoint);
      minDist = min(minDist, d);
    }
  }
  return minDist;
}
```

The KEY property: at a wrap event, `p` shifts by integer-cell vector V. `floor(p + V) = floor(p) + V` (integer-cell-only shift). `fract(p + V) = fract(p)` (fractional part unchanged). The 9-cell neighbour sweep visits the SAME feature points (same `mod(cell + offset, N)` results because V mod N = 0 ‚Äî V is exactly 4 cells, N is exactly 1024 m / 1024 m = 1 in noiseUv units when we set the period correctly). Therefore `worleyPeriodic(p + V) == worleyPeriodic(p)` at every wrap event ‚Äî analytical equality, not a magnitude bound.

This is the architectural property FOG-20 captures. **The wrap injects ZERO discontinuity in the noise output.**

### Step 4 ‚Äî Period alignment invariant

For Worley to produce zero discontinuity, the toroidal period `N` (in noiseUv space) MUST be an integer divisor of the wrap shift in the same space:
```
wrapShift_noiseUv = kPocFogIntegerWrapPeriodMeters / kPocFogNoiseTilePxMeters = 4096 / 1024 = 4
```
N must satisfy `wrapShift_noiseUv % N == 0`. Choices:
- `N = 1` cell ‚Äî feature points repeat every 1024 m; visually monotonous (all cells look identical). Reject for aesthetics.
- `N = 2` cells ‚Äî 2048 m repeat; still monotonous at typical viewport scales. Reject.
- `N = 4` cells ‚Äî 4096 m repeat; matches the wrap period exactly. **THIS IS THE LOCKED CHOICE.** Repeats every wrap, but the wrap is rare (4096 m at z=15 mpp ‚âà 3.16 ‚Üí 1295 m on screen viewport √∑ 12 raw px/sec = ~108 sec between wraps; aesthetically the repetition is invisible because it equals the wrap cadence).
- `N = 8` or higher ‚Äî would require `wrapShift_noiseUv % N == 0` to fail (4 % 8 ‚âÝ 0). Reject ‚Äî break the invariant.

LOCKED: Worley periodic with N = 4 cells (= 4096 m per axis = exactly `kPocFogIntegerWrapPeriodMeters`).

### Step 5 ‚Äî FBM-of-Worley vs single-octave Worley (aesthetic axis only)

Plan 03.1-14's fbm3 produced 3-octave FBM. To preserve aesthetic depth, Plan 03.1-16 may stack 3 octaves of Worley periodic:
```
fbmWorley(p) = 0.5 ¬∑ worley(p)
             + 0.25 ¬∑ worley(p ¬∑ M_1)        // No additive offset to keep period alignment
             + 0.125 ¬∑ worley(p ¬∑ M_1 ¬∑ M_2)
```

CRITICAL: each octave's input shift at a wrap MUST also be integer-multiple of N. With `M_1 = 2¬∑I` (NOT 2.03¬∑I ‚Äî the 0.03 jitter that gradient-FBM uses to break axial alignment is NOT needed for Worley because Worley's feature-point jitter already breaks axial alignment), Octave 2 sees a wrap shift of `M_1 ¬∑ V = (-8, 0, 0)` which is `(-8 mod 4, 0, 0) = (0, 0, 0)` in cells with period N=4 ‚Äî bit-identical at the wrap. Same for Octave 3 with `M_2 = 2¬∑I`: `M_1 ¬∑ M_2 ¬∑ V = (-16, 0, 0)` mod 4 = 0.

LOCKED for FBM-of-Worley path: `M_1 = M_2 = 2¬∑I` (integer scaling); jitter offsets V_1, V_2 must be integer-multiple of N (use `(0, 0, 0)` or `(N, 0, 0)` etc.). The aesthetic offsets gradient-FBM had (`(13.7, 7.3, 5.1)`, `(-11.1, 17.9, 3.3)`) MUST be replaced with integer offsets to preserve the wrap invariance.

Single-octave Worley is also acceptable if aesthetic A/B at /sanity finds it sufficient (Worley naturally has more visible structure than value-noise; one octave may suffice). Planner records the choice in Task 1 commit.

### Step 6 ‚Äî fp32 precision at typical worldMeters values

Worldmeters bounded under 4097 m. noiseUv = worldMeters / 1024 m bounded under ~4. fp32 ULP at noiseUv = 4 is ‚âà 5e-7 ‚Äî negligible. The 9-cell neighbour sweep + distance computation stays in the safe fp32 envelope at all gesture conditions.
